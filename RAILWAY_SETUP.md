# Настройка проекта на Railway

## Шаги для деплоя на Railway

### 1. Подключение репозитория
1. Перейдите на https://railway.com/new/github
2. Выберите репозиторий `kurganovnik88/Vetlan`
3. Railway автоматически определит Python проект

### 2. Настройка переменных окружения

В настройках проекта Railway добавьте следующие переменные окружения:

#### Обязательные переменные:
```
BYBIT_API_KEY=ваш_api_ключ
BYBIT_API_SECRET=ваш_api_секрет
BYBIT_ENV=testnet  # или mainnet для продакшена
```

#### Telegram уведомления (опционально):
```
TELEGRAM_TOKEN=ваш_telegram_токен
TELEGRAM_CHAT_ID=ваш_chat_id
```

### 3. Настройка сервиса

Railway автоматически:
- Определит Python проект
- Установит зависимости из `requirements.txt`
- Запустит команду из `Procfile`: `python run_strategy.py`

### 4. Мониторинг

- Логи доступны в панели Railway
- Приложение будет автоматически перезапускаться при сбоях
- Можно настроить алерты в настройках проекта

### 5. Важные замечания

⚠️ **Безопасность:**
- Никогда не коммитьте `.env` файл (он уже в `.gitignore`)
- Используйте переменные окружения Railway для секретов
- Для тестирования используйте `BYBIT_ENV=testnet`

⚠️ **Производительность:**
- Railway может приостанавливать бесплатные проекты при неактивности
- Для постоянной работы рассмотрите платный план
- Или используйте cron job для периодического запуска

### 6. Проверка работы

После деплоя проверьте логи в Railway:
- Должно появиться сообщение "Запущена стратегия"
- Проверьте подключение к Bybit
- Убедитесь, что Telegram уведомления работают (если настроены)

## Структура файлов для Railway

- `Procfile` - команда запуска
- `requirements.txt` - зависимости Python
- `runtime.txt` - версия Python
- `railway.json` - конфигурация Railway (опционально)
